🔑 How to get Razorpay API Keys
Follow these steps once — then paste your keys below.
1
Sign up or log in at dashboard.razorpay.com
2
Go to Settings → API Keys → Generate Test Key (for development) or Generate Live Key (for production)
3
Copy the Key ID (starts with rzp_test_ or rzp_live_) and Key Secret
4
Paste both below and click Save Keys. Then restart the Django server.
5
For automatic 80/20 split payments, enable Route in Razorpay Dashboard → Products → Route

🛠️ Common Errors & Fixes
Reference guide for troubleshooting Razorpay issues.
❌ "Network error. Please try again."
Django server cannot reach api.razorpay.com. Test with: curl https://api.razorpay.com/v1/orders -u KEY_ID:KEY_SECRET. Usually a firewall or proxy issue on the server. Check internet access on the machine running Django.
⚠️ UnicodeDecodeError when saving keys
Fixed in this version — the .env file is now always read and written with encoding="utf-8". If you still see this, delete the .env file and let the server create a fresh one, then re-enter your keys.
⏳ "Route linking pending" for employee
Employee saved bank details before Razorpay was configured. Ask them to open Bank Details and click Save again — it retries the Razorpay contact + fund account creation automatically.
🔑 BadRequestError / SignatureVerificationError
Wrong key mode — test keys (rzp_test_) won't work with live orders and vice versa. Use Test Connection above to verify. Also check that Key ID and Secret are not swapped.
🔗 Transfer failed / Route not working
Enable Route in Razorpay Dashboard → Products → Route. Also ensure the employee's fund account is verified by Razorpay. Payments still complete normally — only the automatic split to employee is skipped if Route isn't set up.
